# Aesthetic Shadowing
## あなたの審美眼を、AIに継承させる

**フォトグラファーの感性を30枚の判断から学習し、2500枚を自律的にセレクトするAIエージェント**

> 「この写真フォルダをセレクトして」——その一言で、あとは全部やっておきます。

---

## 何ができるか

イベント撮影が終わった。2500枚の写真が手元にある。Lightroomに取り込む前に、まず「選別」しなければならない。この作業が——毎回、数時間かかる。

Aesthetic Shadowingは、その仕事を代行する。

| あなたがすること | エージェントがすること |
|---|---|
| `/photo-selector` と打つ | 撮影意図（運動会・学校PR・結婚式など）を確認する |
| 「運動会の写真です」と答える | 技術的失敗カット（ピンボケ・白飛び・黒潰れ）を自動除外する |
| 30枚にレーティングをつける | その判断からあなたの審美眼を学習する |
| 待つ | 残り2470枚を自律的に採点し、XMPサイドカーをLightroomに書き出す |

**人間の関与時間：常に約6分。**
30枚でも3000枚でも、変わらない。

---

## なぜこれが革新的か

### 従来のAI写真選別との違い

```
従来のAI
  技術品質（シャープネス・露出）のみで判定
  → 「ちゃんと撮れているが面白くない写真」を選ぶ
  → 写真家の感性は完全に無視される

Aesthetic Shadowing
  技術スコア × 審美眼（学習済み） × 撮影意図（文脈）の三重評価
  → 「運動会なら躍動感を、学校PRなら笑顔を」と目的に応じて選ぶ
  → 写真家ごとの「好み」が蓄積され、精度が上がっていく
```

### スケーラビリティの革命

最初の設計（トーナメント方式）では、3000枚で**5.6時間のユーザー拘束**が必要だった。
これはエージェントではなく、人間を縛る装置だった。

設計を全面的に見直した。

```
旧設計: ユーザーが全対戦に立ち会う → 枚数 × 時間
新設計: 20対戦でスタイルを学習 → あとはAIが自律実行
```

| 指標 | 旧設計 | 新設計 |
|------|--------|--------|
| ユーザー拘束時間（3000枚） | **338分** | **6分** |
| APIコール数 | 44,850回 | 3〜5回 |
| コスト削減率 | — | **99.99%** |

### なぜ「自律性」にこだわるのか

このプロジェクトは、技術の問題ではなく**時間の問題**から生まれた。

エンジニアとして働きながら、子育てをしながら、写真を撮り続けている。
イベント後の選別作業に費やす数時間は——家族と過ごせたはずの時間だ。
「家族との時間を豊かにするために、技術で解決する」——これが出発点だった。

だから、人間を拘束するソリューションは、答えではない。

## 技術的な仕組み

4ステージ構成のハイブリッドパイプライン。LLMを使うのは必要な場面だけ。

```
┌─────────────────────────────────────────────────────────────────┐
│  /photo-selector と打つ                                          │
└────────────────┬────────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 0: 意図理解（LLM）                                        │
│  撮影シーンと目的を把握 → 選定基準のベースラインを構成             │
└────────────────┬────────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 1: 技術フィルタ（ローカル・Python/OpenCV）                 │
│  ラプラシアン分散でピンボケを検出                                  │
│  ヒストグラムで白飛び・黒潰れを検出                               │
│  → 救済不能なカットのみ除外（過剰除外はしない）                   │
└────────────────┬────────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 2: シーングループ化（ローカル・EXIF + pHash）              │
│  撮影時刻・視覚的類似度でシーンを塊として認識                     │
│  「最初の1枚」「最後の1枚」に撮影意図ボーナスを付与               │
└────────────────┬────────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 3: 審美眼学習（LLM・ユーザー参加6分）                      │
│  代表ペア20対戦でユーザーの選好を学習                             │
│  → スタイルルールを言語化・構造化                                 │
└────────────────┬────────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 4〜6: 全件自動採点 → 出力（LLM + ローカル）               │
│  学習済みスタイルルールで残り全カットを自律ランキング              │
│  信頼度の低いグループのみスポットチェックをユーザーに提示          │
│  → XMPサイドカーをLightroomに直接書き出し                        │
└─────────────────────────────────────────────────────────────────┘
```

**技術スタック**

- ローカル処理: Python, OpenCV, Rawpy, ExifTool, imagehash
- LLM: Claude（マルチモーダル）
- 出力: XMP Sidecarファイル（Adobe Lightroom連携）
